A modern native language built for humans and coding agents.

AvaLang is an independent open-source programming language and compiler project. The goal is to build a clean native compiler from first principles, with strong diagnostics, structured compiler output, and a path toward self-hosting.

Why AvaLang

A compiler project for the LLM and coding-agent era.

Human readable

AvaLang is designed to be explicit, readable, and teachable. The public implementation will be written as a tutorial so readers can understand every compiler stage.

Agent friendly

The compiler will prioritize stable error codes, precise source spans, JSON diagnostics, and tutorial-linked examples so coding agents can interact with it reliably.

Native by default

AvaLang is planned as a native compiled language. The early compiler will target macOS and Linux from the beginning.

Design goals

Small steps, serious architecture.

AvaLang starts with a lean C++ bootstrap compiler, then gradually moves higher-level behavior into .ava source files. The long-term goal is a self-hosting compiler written in AvaLang.

  • The command-line tool is avalang.
  • Source files use the .ava extension.
  • Every tutorial checkpoint will be testable and tagged.
  • Diagnostics are treated as a core compiler interface.
  • macOS and Linux support are planned from the beginning.
